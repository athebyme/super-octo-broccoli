# Брейншторм: новые фичи для WB Seller Platform

> Дата: 2026-03-11
> Основано на анализе текущего функционала, WB API спецификаций и реальных данных аккаунта

---

## Приоритет 1: Высокая ценность + быстрая реализация

### 1. AI-автоответы на отзывы и вопросы
**Проблема:** 1097 отзывов в архиве, многие без ответа продавца. Ответы на отзывы повышают лояльность и рейтинг карточки.

**Что делаем:**
- Загрузка неотвеченных отзывов через `GET /api/v1/feedbacks?isAnswered=false`
- AI генерирует персонализированный ответ на основе текста отзыва, оценки, названия товара
- Для позитивных (4-5 звёзд) — благодарность + рекомендация сопутствующих товаров
- Для негативных (1-3 звёзды) — извинение + предложение решения + контакт
- Модерация: продавец видит сгенерированный ответ и может отредактировать перед отправкой
- Автоответ через `PATCH /api/v1/feedbacks`
- Шаблоны ответов: `POST /api/v1/templates`
- Аналогично для вопросов: `GET /api/v1/questions`, `PATCH /api/v1/questions`

**API эндпоинты (все доступны):**
- `GET /api/v1/feedbacks` — получение отзывов ✅ протестировано
- `GET /api/v1/questions` — получение вопросов ✅ протестировано
- `PATCH /api/v1/feedbacks` — ответ на отзыв
- `PATCH /api/v1/questions` — ответ на вопрос
- `GET /api/v1/feedbacks/count` — счётчик ✅ протестировано (1097 отзывов)

**Оценка:** 3-5 дней разработки

---

### 2. Дашборд отзывов и репутации
**Проблема:** Нет единого места для мониторинга репутации магазина.

**Что делаем:**
- Сводная панель: средний рейтинг, распределение по звёздам, динамика
- Список товаров с худшим рейтингом (уже есть nmRating в карточках)
- Облако тегов из негативных отзывов (AI-анализ sentiment)
- Алерты на новые негативные отзывы (1-2 звезды)
- Время ответа на отзывы (SLA метрика)
- Экспорт отзывов в CSV для внешнего анализа

**Оценка:** 2-3 дня

---

### 3. Аналитика возвратов и отмен
**Проблема:** В данных заказов есть поля `isCancel`, `cancelDate` — но нет аналитики по ним.

**Что делаем:**
- Процент отмен по товарам, категориям, регионам
- Тренд отмен во времени (растёт/падает)
- Топ-10 товаров с наибольшим % отмен
- Корреляция отмен с ценой, скидкой, SPP
- Анализ причин: привязка отмен к конкретным складам (проблемы доставки?)
- Рекомендации: какие товары стоит убрать/оптимизировать

**Данные уже доступны:** orders API содержит `isCancel`, `cancelDate`, `regionName`, `warehouseName` ✅

**Оценка:** 2-3 дня

---

### 4. Региональная аналитика продаж
**Проблема:** Данные о регионах есть в каждом заказе/продаже, но никак не визуализируются.

**Что делаем:**
- Карта продаж по федеральным округам и регионам
- Топ регионов по выручке / количеству заказов
- Тепловая карта: какие товары покупают в каких регионах
- Средний чек по регионам
- SPP (скидка WB) по регионам — где WB даёт больше скидок?
- Рекомендации: в каких регионах расширять ассортимент

**Данные:** В каждом заказе/продаже есть `oblastOkrugName`, `regionName`, `countryName` ✅

**Оценка:** 3-4 дня

## Приоритет 2: Средняя ценность, среднее время

### 5. Складская аналитика и оптимизация
**Проблема:** Товары распределены по складам (Коледино, Казань, Невинномысск и др.), но нет анализа эффективности.

**Что делаем:**
- Остатки по складам с визуализацией (текущие данные stocks API ✅)
- Скорость продаж по складам (warehouse turnover rate)
- Рекомендации по перемещению товара на более эффективные склады
- Алерты: товар заканчивается на складе (< N штук)
- «Мёртвые» остатки: товары, которые не продаются > 30 дней
- FBS vs FBW сравнение эффективности

**Данные:** stocks API — `warehouseName`, `quantity`, `inWayToClient`, `inWayFromClient`, `isSupply`, `isRealization` ✅

**Оценка:** 4-5 дней

---

### 6. Прогнозирование спроса
**Проблема:** Нет понимания, когда и сколько товара нужно закупить.

**Что делаем:**
- Анализ истории продаж за 90 дней (sales API ✅)
- Простая модель прогнозирования (скользящее среднее, экспоненциальное сглаживание)
- Рекомендуемый объём закупки с учётом сроков поставки
- Сезонность: выявление паттернов по дням недели, месяцам
- Алерт: "Через N дней товар закончится при текущей скорости продаж"
- Визуализация: график прогноза на 30/60/90 дней

**Оценка:** 5-7 дней

---

### 7. Мониторинг конкурентов (через публичный API)
**Проблема:** Нет понимания ценовой позиции относительно конкурентов.

**Что делаем:**
- Парсинг публичных данных WB по аналогичным товарам в категории
- Сравнение цен: позиция твоего товара среди конкурентов
- Отслеживание изменений цен конкурентов
- Средняя цена в категории, медиана
- Рекомендации по оптимальной цене
- Алерты: конкурент резко снизил цену

**Оценка:** 5-7 дней

---

### 8. Финансовый отчёт с детализацией
**Проблема:** Есть profit calculator, но нет детального финансового отчёта по данным WB API.

**Что делаем:**
- Выгрузка через `GET /api/v5/supplier/reportDetailByPeriod` (еженедельный отчёт WB)
- Детализация: комиссия WB, логистика, штрафы, хранение, удержания
- Реальная маржинальность каждого товара (с учётом всех комиссий)
- P&L отчёт за период
- Сравнение плановой маржи (из pricing engine) vs фактической
- Выявление убыточных товаров

**Оценка:** 4-6 дней

## Приоритет 3: Стратегические фичи

### 9. Динамическое ценообразование
**Проблема:** Цены устанавливаются вручную формулой и не реагируют на рынок.

**Что делаем:**
- Автоматическая корректировка цен на основе:
  - Скорости продаж (растут → можно повысить)
  - Остатков (мало → повысить, много → снизить для ускорения)
  - Конкуренции (мониторинг конкурентов)
  - Дня недели / времени суток
- Правила: min/max цена, максимальное изменение за период
- A/B тестирование цен
- Лог всех автоматических изменений
- Режим «симуляция» — показать что бы изменилось, не применяя

**Оценка:** 7-10 дней

---

### 10. AI-анализ sentiment отзывов
**Проблема:** 1097 отзывов — ручной анализ невозможен.

**Что делаем:**
- Массовая загрузка всех отзывов
- AI-классификация: позитивный / нейтральный / негативный
- Извлечение тем: качество, упаковка, доставка, описание, цена
- Тренды sentiment по времени
- Привязка к конкретным товарам: "У товара X резко вырос негатив по теме упаковка"
- Автоматические рекомендации: "Улучшить упаковку товара X" / "Обновить описание Y"

**Оценка:** 3-5 дней (AI инфраструктура уже есть)

---

### 11. Система рекламных кампаний
**Проблема:** WB имеет API для рекламы (`/adv/v1/*`), но в платформе это не используется.

**Что делаем:**
- Создание/управление рекламными кампаниями через API
- Мониторинг расходов и ROI рекламы
- Автоматический расчёт ACOS (Advertising Cost of Sales)
- Рекомендации: какие товары рекламировать (высокая маржа + хорошие отзывы)
- Автостоп кампании при превышении бюджета
- Связка: реклама → продажи → прибыль (сквозная аналитика)

**Оценка:** 7-10 дней

---

### 12. Мультиселлер / командная работа
**Проблема:** Платформа односеллерная, а бизнес может масштабироваться.

**Что делаем:**
- Несколько WB аккаунтов в одной платформе
- Роли: владелец, менеджер, аналитик (только чтение)
- Общий каталог поставщиков для нескольких селлеров
- Сводная аналитика по всем аккаунтам
- Лог действий каждого пользователя (уже есть AdminAuditLog)

**Оценка:** 7-10 дней

---

### 13. Автоматические уведомления и алерты
**Проблема:** Система уведомлений есть, но нет умных триггеров.

**Что делаем:**
- Telegram-бот для уведомлений (или email)
- Триггеры:
  - Новый негативный отзыв (1-2 звезды)
  - Товар заканчивается на складе
  - Резкое падение продаж товара
  - Подозрительное изменение цены
  - Новый заблокированный/скрытый товар
  - Заказ отменён (для FBS)
- Настраиваемые правила: порог, частота, каналы

**Оценка:** 3-5 дней

---

### 14. Возвраты покупателями
**Проблема:** WB API имеет эндпоинт для возвратов, но он не интегрирован.

**Что делаем:**
- Список заявок на возврат: `GET /api/v1/claims` (Communication API)
- Статистика возвратов по товарам
- Причины возвратов (AI-анализ текста заявки)
- Процент возвратов vs продаж
- Корреляция возвратов с описанием товара (несоответствие ожиданиям)
- Рекомендации: улучшить описание/фото для уменьшения возвратов

**Оценка:** 3-4 дня

## Технические заметки

### Доступные данные (протестировано с реальным токеном):
- **Feedbacks:** 1097 отзывов, включая productDetails, rating, text, photos
- **Questions:** 394 вопроса
- **Orders:** полная история с регионами, складами, отменами, SPP
- **Sales:** продажи с finishedPrice, forPay, paymentSaleAmount
- **Stocks:** остатки по всем складам с количеством и движением

### Ключевые поля данных для новых фичей:
```
Orders: date, regionName, oblastOkrugName, warehouseName, totalPrice,
        discountPercent, spp, finishedPrice, isCancel, cancelDate, nmId
Sales:  date, regionName, warehouseName, totalPrice, spp, forPay,
        finishedPrice, priceWithDisc, paymentSaleAmount
Stocks: warehouseName, quantity, inWayToClient, inWayFromClient,
        quantityFull, Price, Discount, isSupply, isRealization
Feedbacks: id, text, pros, cons, productValuation, createdDate,
           productDetails (nmId, productName, brandName), userName, photoLinks
```
